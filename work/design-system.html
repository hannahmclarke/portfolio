Component Library

A cross-product UI component library, built web-components-first and shipped as React 18, React 19 (SSR) and styling packages. I'm one of its principal engineers — owning much of the release engineering, CI/CD, testing infrastructure and developer tooling, alongside net-new component work.

What it is

The component library is the source of truth for UI across multiple products. Components are authored once as Stencil web components, then auto-generated into framework packages: a React 18 wrapper set, a React 19 package with server-side-rendering support for the Next.js App Router, plus shared SCSS styles and design tokens synced from Figma. Everything is published from an npm-workspaces monorepo to a private package registry, with Storybook deployed to a CDN as living documentation.

My role

I'm the second-largest contributor to the system and its de-facto release and build owner. My work clusters in four areas: release & build engineering, testing infrastructure, developer/AI tooling, and component development. The points below are drawn from my own authored, merged pull requests — not the team's aggregate output.

What I built

Migrated the entire monorepo from pnpm to npm workspaces

Removed the pnpm lockfile and workspace config across ~50 files and rewired every build, clean and Stencil script — a foundational change to how the whole system installs and builds.

Stood up CI/CD on Azure Pipelines and retired the legacy Jenkins setup

Built the build/test/deploy pipelines, configured CDN deploys across environments and the automated testing pipeline, then removed the old Jenkins deployment entirely.

Introduced visual-regression testing with Chromatic

Wired Chromatic story parameters across ~17 components and integrated it into the pipeline, so visual diffs are caught automatically before they reach consuming products.

Built the React end-to-end test runner and Storybook interaction testing

Added the Puppeteer/Jest e2e harness for the React package, migrated legacy tests, and later added React component testing in Storybook plus shared testing helpers.

Built the Radio Button & Radio Button Group components end-to-end

The full stack: Stencil source, generated React wrappers, SCSS module and shared form-input styles, unit + e2e tests, and Storybook stories — plus a reusable story template for toggling component attributes.

Owned the release cadence and Storybook platform

Run the recurring develop→main release process — changelog and version management, sync automation — and led Storybook upgrades (through v10.x) with the docs deploy and production hotfixes. Also migrated the Icon component to SVG-based rendering.

Built the AI developer-tooling layer

Set up the team's AI coding integration and shared context system, authored reusable upgrade "skills" and a slash-command for consuming teams, added Figma-MCP setup, and generated a machine-readable component manifest for AI-agent consumption.

Team context

This is a large, multi-engineer system; the components catalogue as a whole is the work of the team. Because the repo squash-merges, release commits I author often bundle teammates' component PRs — so I've deliberately listed only work I authored and verified (build/release, testing, tooling, and the radio-button components), and credited the broader component catalogue to the team.
