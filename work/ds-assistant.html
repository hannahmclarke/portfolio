Design System AI Assistant

An AI Slack bot and autonomous "curator" agent that answers design-system questions from across the team's tools, backed by a curated knowledge base exposed over MCP. I designed and built it end-to-end — solo.

What it is

A Slack-native assistant that answers design-system questions by reasoning over multiple live sources — design docs, Figma, Storybook, Confluence and the component codebase — and replies in-thread. A separate curator agent reads the codebase, diffs it against a curated markdown knowledge base, and drafts updates for human review. That knowledge base is also exposed as an MCP server so it can be consumed directly inside an IDE. It runs as a single deployed service with its own analytics.

My role

This was a solo build: I authored 124 of 126 commits (the other two are an automation account and the curator agent I built committing its own draft). Architecture, integrations, the agent, deployment and analytics are all mine. It was developed against tickets and merged through pull requests even as a single contributor.

What I built

The Slack bot, end-to-end

A Slack Bolt app (Socket Mode) running a Claude tool-use loop: a cheap classifier first guards scope, then the model works across the data-source tools and replies in-thread with a rolling conversation window. Plus a three-tier feedback mechanism persisted for later analysis.

Six live data-source integrations

Tools the model can call to search design docs, the Figma API, Storybook (multi-framework), Confluence, the icon library, and to search/read the component codebase — each with its own tests.

The autonomous curator agent

Built on the Claude Agent SDK with knowledge-base reader/writer tools and a draft-queue review flow — it reads the codebase, detects drift from the knowledge base, and proposes edits, with nothing auto-merging without human review.

The knowledge base & seed pipeline

A curated markdown knowledge base plus a seed script that imports generated component documentation — the substrate both the bot and the MCP server read from.

Analytics & monitoring

A logging layer with a SQLite store, a lightweight classifier for categorising questions, CLI reports, and an automated weekly Slack digest — so usage and gaps are measurable.

Deployment & a self-running curator pipeline

Deployed to Azure App Service, plus an event-driven pipeline that runs the curator and opens its own knowledge-base-update pull requests automatically — the agent maintaining its own source of truth.

Honest status

The Slack bot, curator agent, data-source integrations, analytics and deployment are built and running. The full MCP server and a couple of later phases were planned/partial at the time of writing — I've described those as roadmap rather than shipped. This was also a project where I leaned on AI tooling under my own direction; the engineering decisions and integration are mine.
